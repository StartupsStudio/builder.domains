# Free Domains for Builders: How builder.domains Removes Friction from Launching

Every builder has a graveyard of ideas that never launched. Not because they were bad ideas, but because the friction between idea and execution was too high. Domain registration fees—$10-15 per domain—create a surprising psychological barrier that kills experimentation.

## The $15 Decision Paralysis

Consider the builder mindset when evaluating a new idea:

**With paid domains:**
```
Idea: Build a tool to X
  ↓

  ↓

  ↓

  - Is this idea worth $15?
  - Will I actually finish this?
  - What if nobody uses it?

  ↓
Decision paralysis
  ↓
Idea abandoned
```

**With free domains:**
```
Idea: Build a tool to X
  ↓
Claim domain (free, instant)
  ↓
Start building
```

The difference is profound. Free removes the decision point entirely.

## The Economics of Experimentation

### Traditional Domain Economics

A builder exploring 10 ideas per year:

- **Registration**: 10 × $15 = $150
- **Most don't launch**: ~7 abandoned = $105 wasted
- **Annual renewal**: Even failed projects = $150/year ongoing
- **Total 3-year cost**: $450-600

**Perverse incentive**: Keep paying for failed projects to "preserve" the domain name, or let it expire and feel like you wasted money.

### Free Domain Economics

Same builder with builder.domains:

- **Registration**: 10 × $0 = $0
- **Experiment freely**: No sunk cost fallacy
- **Release unused**: No annual renewal pressure
- **Upgrade winners**: Only pay for successful projects

**Aligned incentive**: Experiment liberally, upgrade when you have product-market fit.

## Integration with Builder Workflows

### With GitHub

```yaml
# .github/workflows/preview.yml
name: Preview Deployment
on: [pull_request]

jobs:
  preview:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Claim preview domain
        run: |
          DOMAIN=$(builder-domains claim "pr-${{ github.event.number }}-$(git rev-parse --short HEAD)")
          echo "PREVIEW_DOMAIN=$DOMAIN" >> $GITHUB_ENV

      - name: Deploy
        run: vercel --yes
        env:
          VERCEL_DOMAIN: ${{ env.PREVIEW_DOMAIN }}

      - name: Comment PR
        run: |
          gh pr comment ${{ github.event.number }} \
            --body "Preview: https://${{ env.PREVIEW_DOMAIN }}"
```
